# AStoryForge

> 让你用 AI 把故事搭出来。

AStoryForge 是一个面向小说创作者的 AI 原生叙事引擎。
创作者负责"搭骨架"——定义事件、人物、世界观、叙事顺序；
AI 负责"填血肉"——在多智能体模拟沙盘里推演、评估、生成。

## 项目简介

通用 AI 在长篇创作上容易"写着写着忘了前面"或"角色人设崩塌"。
AStoryForge 用以下四个机制把创作权交回创作者手中：

| 问题 | 解决方案 |
|------|----------|
| 长文逻辑混乱 | 事件有向图显式定义因果，AI 只在节点内填充 |
| 角色人设崩塌 | 模拟沙盘由"演员智能体"扮演角色，"观察者智能体"实时评估 |
| 大纲与成文脱节 | 编译管线把 JSON 节点结构化为 MD 大纲，AI 严格对齐生成 |

> 把故事当成一张可拖拽的事件图，让 AI 在你定义的"沙盘"里演给你看，再按需展开成小说。

## 核心特性

### 1. 事件有向图 + 实体库

故事被建模为两个正交维度：

- **事件（Event）** —— 故事最小叙事单元，有入边/出边/子事件/参与者
- **实体（Entity）** —— 故事中的"演员"，记录自身状态弧光

三种正交关系：

| 关系 | 表达 | 回答的问题 |
|------|------|------------|
| 因果 | `in_edges[]` / `out_edges[]` | "为什么发生" |
| 包含 | `sub_events[]` | "属于哪个部分" |
| 参与 | `participants[]` | "谁在场，什么状态" |

### 2. spine 主链（叙事顺序控制）

把"事件真实发生的时间"和"读者看到的顺序"解耦：

- **故事时间轴**：由 `time` + 边关系决定
- **叙事链（spine）**：`project.json` 里的有序数组，可自由重排

倒叙、插叙、平行叙事、时间跳跃，全部通过拖拽 spine 完成。

### 3. 模拟沙盘（多智能体推演）

放入角色、设定初始条件，角色自主行动，第三方观察者实时评估：

```
角色 A 输出行动 → 观察者评估 → 合理？→ 更新状态 → 下一轮
                              ↘ 不合理？→ 自动回退或人工介入
```

支持四种推演类型：

- `continue`：续写后续情节
- `fill_detail`：在父事件内部细化过程
- `load_from_raw`：从原始文本中提取并结构化信息
- `between`：填补两个事件之间的空白（兼容模式）
- `inside`：细化父事件内部过程（兼容模式）

支持分级评估：

- `approved` —— 通过，继续
- `flagged` —— 基本通过，附问题提示
- `rejected` —— 严重不合理，触发回退

并配有硬约束 / 软约束 / 世界观规则三级规则体系。

### 4. 编译管线（从 JSON 到小说正文）

```
nodes/*.json + chars/*.json + project.json
                     ↓
       编译管线（加载 → 展开 → 验证 → IR → 生成）
                     ↓
        结构化 MD 字节码（完整 Markdown 大纲）
                     ↓
          DeepSeek（逐事件展开，填充血肉）
```

- 大纲生成**不需要大模型**，零成本可无限次调整
- 编译阶段自动检测：循环引用、时间矛盾、因果断裂

## 架构概览

四层架构：

```
┌─────────────────────────────────────────────────────────────┐
│                    第一层：输入与梳理                         │
│   用户上传任何文本 → 小型模型提取 → 结构化设定/人物/世界观      │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    第二层：叙事链编辑                         │
│   事件节点可拖拽 → spine 数组定义叙事顺序 → 因果事件网络        │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    第三层：模拟推演                           │
│   角色扮演式推演 → 第三方观察者评估 → 用户审查 → 锁定大纲       │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    第四层：编译管线 + AI 生成                 │
│   JSON 节点 → 编译为结构化 MD → DeepSeek 逐事件展开为正文      │
└─────────────────────────────────────────────────────────────┘
```

后端技术栈：

- **语言**：Go 1.25+
- **Web 框架**：Gin
- **AI 客户端**：[go-openai](https://github.com/sashabaranov/go-openai) 兼容 OpenAI / LM Studio / Ollama 等 OpenAI 协议端点
- **MCP**：[mcp-go](https://github.com/mark3labs/mcp-go) 用于工具协议
- **日志**：[logrus](https://github.com/sirupsen/logrus)
- **配置**：[go-yaml](https://github.com/goccy/go-yaml)

## 快速开始

### 环境要求

- 一个兼容 OpenAI 协议的模型端点（OpenAI / DeepSeek / LM Studio / Ollama 等）

### 克隆与构建

```bash
git clone https://github.com/your-name/AStoryForge.git
cd AStoryForge/Server
go mod download
go build -o AStoryForge .
```

### 初始化配置文件

```bash
./AStoryForge -i
```

会在当前目录生成 `settings.yaml` 模板。

### 启动服务

```bash
./AStoryForge -s settings.yaml
```

默认监听 `settings.yaml` 中 `WebSocket.Host` 配置的地址。

### 命令行参数

| 参数 | 短参 | 说明 |
|------|------|------|
| `--initSetting` | `-i` | 初始化配置文件 |
| `--setting` | `-s` | 指定配置文件路径，默认为 `settings.yaml` |
| `--connectTest` | `-c` | 连通性测试，验证模型端点是否可达 |
| `--doc` / `--help` | - | 显示帮助文档 |

## 配置说明

`settings.yaml` 顶层字段：

```yaml
run_mode: develop   # develop | release

Gin: {}             # Web 框架相关配置

log:
  app: AStoryForge
  dir: logs
  log_level: debug  # debug | info | warn | error

ai:                 # 智能体列表（可注册多个）
  - enable: true
    name: 角色名
    prompt: "人格提示词"
    model: qwen/qwen3.5-9b
    temperature: 0.7
    max_tokens: 150
    host: http://192.168.127.1:1234/v1
    apiKey: ""
    apiType: openai
```

`ai` 列表里的每一项都是一个独立的智能体。沙盘推演时会按 `actor_agent_indexes` / `observer_agent_indexes` 把这些智能体分配为"演员"或"观察者"。

## 关于 AI 辅助编程的态度

> 目前为 demo 的原型验证阶段，对 AI 生成的代码保持保守态度。

- **拒绝全程 vibe coding**：拒绝不假思索地把整个项目交给 AI 自由发挥。
- **必须思路清晰**：在每一段代码动手前，自己先想清楚要解决什么问题、边界在哪里。
- **必须明白 AI 在做什么**：对 AI 给出的每一段代码，能讲清楚它在做什么、为什么这样写、副作用是什么。
- **评审是底线**：AI 写完的代码和人类写的代码走同样的评审标准。
